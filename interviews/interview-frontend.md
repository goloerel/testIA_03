Role: Frontend Developer (Vanilla JS/CSS)

Interviewer: How do you approach CSS naming and file organization?

User: I prefer single hyphens - for naming; no underscores _ or double symbols. I want one main.css for general styles, though specific component files are okay if needed.

Interviewer: How do you handle JavaScript state and data flow?

User: I prefer passing data directly between functions rather than using a global store.

Interviewer: What are the requirements for API calls?

User: The focus is on vanilla JS patterns that are easy to maintain and reuse.
